# 实时语音面试

## 功能概述

实时语音面试链路采用三段式处理：

1. 候选人语音 -> ASR 转写
2. 转写文本 -> 决策层 + LLM 生成
3. 生成文本 -> TTS 合成并分片下发音频

该模式保证业务流程由后端控制，避免端到端模型失控。

## 关键特性

- 后端统一控轮：问候、主问题、追问、结束均由 `TurnPlanner` + `RealtimeTurnOrchestrator` 决定
- 决策层可回退：JSON 解析失败/超时/API 错误时自动 fallback 到 legacy plan
- 语音输出可观测：每轮都有 `response.created`、`response.audio.delta`、`response.done`

## 协议

### 客户端 -> 后端

```json
{"type":"audio","audio":"<base64 pcm16>"}
{"type":"end_turn"}
{"type":"no_response_timeout"}
```

### 后端 -> 客户端

```json
{"type":"response.created","response":{"id":"...","status":"in_progress"}}
{"type":"response.audio_transcript.delta","response_id":"...","delta":"..."}
{"type":"response.audio.delta","response_id":"...","audio":"<base64 pcm16>"}
{"type":"response.done","response":{"id":"...","status":"completed"}}
{"type":"error","error":{"code":"...","message":"..."}}
```

## 状态推进

- 候选人输入完成后：`commit -> transcription -> decision -> generate -> tts -> done`
- 仅 `turn.status=COMPLETED` 才会推进业务状态。
- `closing` 轮完成后，发送 `interview.natural_end` 给前端。

## 主要配置

- `ARK_ASR_MODE`：ASR 接入模式
- `ARK_LLM_MODEL`：统一文本模型
- `ARK_TTS_VOICE`：TTS 音色
- `REALTIME_DECISION_*`：决策层策略参数
