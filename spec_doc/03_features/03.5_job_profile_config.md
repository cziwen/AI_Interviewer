# 岗位配置管理

## 📝 概述

岗位配置（JobProfile）系统允许 HR 为不同岗位上传专属的题库和岗位描述（JD），实现个性化面试内容。系统支持通过 CSV 文件导入题库，通过 JSON 文件配置 JD 和面试参数。

## 🎯 功能特性

- ✅ **CSV 题库导入**：批量上传面试题目和参考答案
- ✅ **JSON JD 配置**：结构化岗位描述和面试控制参数
- ✅ **动态抽题**：创建面试时随机从题库抽取指定数量的题目
- ✅ **面试参数配置**：主问题数量、追问限制、预期时长
- ✅ **岗位复用**：同一岗位配置可用于多个面试

## 📦 数据模型

### JobProfile 模型

**代码位置**：[backend/app/models/job_profile.py](../../backend/app/models/job_profile.py)

```python
class JobProfile(Base):
    __tablename__ = "job_profiles"

    id = Column(Integer, primary_key=True, index=True)
    position_key = Column(String, unique=True, index=True)  # 岗位唯一标识
    position_name = Column(String, nullable=True)           # 岗位展示名称
    jd_data = Column(JSON)                                  # JD 结构化数据
    question_bank = Column(JSON)                            # 题库列表
    created_at = Column(DateTime, default=datetime.utcnow)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)
```

### jd_data 结构

```json
{
  "responsibilities": "负责后端服务开发和优化，参与系统架构设计",
  "requirements": "精通 Python，熟悉 Django/FastAPI 框架，了解微服务架构",
  "plus": "有大规模系统开发经验，熟悉云原生技术栈",
  "main_question_count": 3,
  "followup_limit_per_question": 1,
  "expected_duration_minutes": 10
}
```

| 字段 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| `responsibilities` | string | 否 | 岗位职责 |
| `requirements` | string | 否 | 任职要求 |
| `plus` | string | 否 | 加分项 |
| `main_question_count` | int | 是 | 主问题数量（从题库抽取） |
| `followup_limit_per_question` | int | 是 | 每个主问题的追问次数上限 |
| `expected_duration_minutes` | int | 是 | 预期面试时长（分钟） |

### question_bank 结构

```json
[
  {
    "question_text": "请介绍一下 Python 的 GIL",
    "reference": "全局解释器锁的概念、影响和规避方法"
  },
  {
    "question_text": "如何设计一个高并发系统？",
    "reference": "负载均衡、缓存策略、数据库优化、异步处理"
  },
  {
    "question_text": "请描述一次线上故障排查经历",
    "reference": null
  }
]
```

| 字段 | 类型 | 说明 |
|-----|------|------|
| `question_text` | string | 问题文本 |
| `reference` | string \| null | 参考答案要点/方向（可选） |

## 📁 文件格式

### CSV 题库格式

**文件名示例**：`backend_engineer_questions.csv`

**格式要求**：
- 编码：UTF-8（支持 UTF-8-BOM）
- 第一行：列头（`question`, `reference`）
- 分隔符：逗号（`,`）

**示例**：

```csv
question,reference
请介绍一下 Python 的 GIL,全局解释器锁的概念、影响和规避方法
如何设计一个高并发系统？,负载均衡、缓存策略、数据库优化、异步处理
请描述一次线上故障排查经历,
什么是 RESTful API 设计原则？,资源导向、HTTP 方法、状态码、版本控制
如何优化 SQL 查询性能？,索引优化、查询计划、分页、避免 N+1 问题
```

**注意事项**：
- `reference` 列可以为空（表示开放题）
- 问题文本不能为空
- 如果问题或参考答案包含逗号，使用双引号包裹
- 支持多行文本（使用双引号包裹）

### JSON JD 格式

**文件名示例**：`backend_engineer_jd.json`

**示例**：

```json
{
  "responsibilities": "负责公司核心业务系统的后端开发和维护，参与系统架构设计和优化，编写高质量、可维护的代码。",
  "requirements": "本科及以上学历，计算机相关专业；3年以上 Python 后端开发经验；精通 Django 或 FastAPI 框架；熟悉 MySQL、Redis 等数据库；了解微服务架构和容器化技术。",

  "main_question_count": 3,
  "followup_limit_per_question": 1,

```

**最小示例**（仅必填字段）：

```json
{
  "main_question_count": 3,
  "followup_limit_per_question": 1,
  "expected_duration_minutes": 10
}
```

## 🔧 API 使用

### 1. 上传岗位配置

**端点**：`POST /api/job-profiles/`

**请求**：`multipart/form-data`

| 字段 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| `position_key` | string | 是 | 岗位唯一标识（如 `backend_engineer`） |
| `position_name` | string | 否 | 岗位展示名称（如 `后端工程师`） |
| `jd_file` | file | 是 | JD JSON 文件 |
| `question_csv` | file | 是 | 题库 CSV 文件 |

**示例**：

```bash
curl -X POST http://localhost:8000/api/job-profiles/ \
  -F "position_key=backend_engineer" \
  -F "position_name=后端工程师" \
  -F "jd_file=@backend_engineer_jd.json" \
  -F "question_csv=@backend_engineer_questions.csv"
```

**响应**：

```json
{
  "position_key": "backend_engineer",
  "position_name": "后端工程师",
  "jd_data": {
    "responsibilities": "...",
    "requirements": "...",
    "main_question_count": 3,
    "followup_limit_per_question": 1,
    "expected_duration_minutes": 10
  },
  "question_bank": [
    {
      "question_text": "请介绍一下 Python 的 GIL",
      "reference": "全局解释器锁的概念、影响和规避方法"
    },
    ...
  ]
}
```

### 2. 列出所有岗位配置

**端点**：`GET /api/job-profiles/`

**响应**：

```json
[
  {
    "position_key": "backend_engineer",
    "position_name": "后端工程师",
    "jd_data": { ... },
    "question_bank": [ ... ]
  },
  {
    "position_key": "frontend_engineer",
    "position_name": "前端工程师",
    "jd_data": { ... },
    "question_bank": [ ... ]
  }
]
```

### 3. 获取单个岗位配置

**端点**：`GET /api/job-profiles/{position_key}`

**示例**：

```bash
curl http://localhost:8000/api/job-profiles/backend_engineer
```

### 4. 更新岗位配置

使用相同的 `POST /api/job-profiles/` 端点，传入相同的 `position_key`：

```bash
curl -X POST http://localhost:8000/api/job-profiles/ \
  -F "position_key=backend_engineer" \
  -F "jd_file=@updated_jd.json" \
  -F "question_csv=@updated_questions.csv"
```

**说明**：
- 如果 `position_key` 已存在，会覆盖原配置
- 未提供 `position_name` 时保留原值

## 🎬 创建面试时使用岗位配置

**代码位置**：[backend/app/api/interviews.py:21-65](../../backend/app/api/interviews.py#L21)

**请求**：

```json
{
  "name": "张三",
  "position_key": "backend_engineer",
  "external_id": "candidate_001",
  "resume_brief": "5年 Python 开发经验，熟悉 Django 和 FastAPI"
}
```

**关键字段**：
- `position_key`：指定使用哪个岗位配置（如果不提供，则使用自动生成题目）

**流程**：

```python
# 1. 查询岗位配置
job_profile = db.query(JobProfile).filter(
    JobProfile.position_key == interview_in.position_key
).first()

# 2. 获取面试参数
main_question_count = job_profile.jd_data.get('main_question_count', 3)

# 3. 从题库随机抽题
bank = job_profile.question_bank
sample_size = min(main_question_count, len(bank))
sampled = random.sample(bank, sample_size)

# 4. 重新编号
questions = [
    {
        "order_index": i + 1,
        "question_text": q["question_text"],
        "reference": q.get("reference")
    }
    for i, q in enumerate(sampled)
]

# 5. 创建面试
db_interview = Interview(
    name=interview_in.name,
    position=job_profile.position_name or position,
    question_set=questions,
    ...
)
```

## 🔍 实际应用场景

### 场景 1：技术岗位（后端工程师）

**题库内容**（20 题）：
- 编程语言基础（5 题）
- 框架与工具（5 题）
- 数据库与缓存（4 题）
- 系统设计（3 题）
- 故障排查与经验（3 题）

**面试参数**：
- `main_question_count`: 3（从 20 题中随机抽 3 题）
- `followup_limit_per_question`: 1（每题最多 1 次追问）
- `expected_duration_minutes`: 10

### 场景 2：管理岗位（产品经理）

**题库内容**（15 题）：
- 产品思维（4 题）
- 需求分析（3 题）
- 项目管理（3 题）
- 数据分析（2 题）
- 沟通协作（3 题）

**面试参数**：
- `main_question_count`: 4
- `followup_limit_per_question`: 2（管理岗位需要更深入交流）
- `expected_duration_minutes`: 15

## 📊 面试参数对实际流程的影响

### main_question_count

**作用**：决定从题库抽取多少道题

**影响**：
- AI 会按顺序问完所有主问题
- 问题数量越多，面试时间越长
- 推荐：初筛 2-3 题，深度面试 4-5 题

### followup_limit_per_question

**作用**：每个主问题后最多追问几次

**影响**：
- `0`：不追问（快速筛选）
- `1`：轻度追问（常规面试）
- `2-3`：深度追问（关键岗位）

**AI 行为**：
- AI 会根据候选人回答质量决定是否追问
- 追问次数用完后，自动进入下一个主问题

### expected_duration_minutes

**作用**：面试预期时长

**影响**：
- **节奏控制**：如果进度落后，AI 会减少追问
- **超时处理**：达到时长后，AI 会自然地结束面试
- **硬超时**：超过预设时长的 20%（或 5 分钟）后，强制结束

**推荐值**：
- 初筛：5-8 分钟
- 常规面试：10-15 分钟
- 深度面试：15-20 分钟

详见：[实时面试功能 - 节奏控制](03.2_realtime_interview.md#4-节奏控制与超时处理)

## 🛠️ 最佳实践

### 1. 题库设计原则

**✅ 推荐**：
- 题目具体、明确
- 包含不同难度层级
- 提供参考答案方向（不是标准答案）
- 题库数量 ≥ 主问题数量 × 3（保证随机性）

**❌ 避免**：
- 过于开放的题目（如"介绍一下你自己"）
- 需要长篇大论的题目
- 过于简单的是/否题

### 2. 参考答案编写

**✅ 推荐**：
```csv
question,reference
如何设计一个高并发系统？,负载均衡、缓存策略、数据库优化、异步处理、限流降级
```

- 列出关键要点（用逗号分隔）
- 提供思考方向而非标准答案
- 帮助 AI 判断回答完整性

**❌ 避免**：
```csv
question,reference
如何设计一个高并发系统？,需要考虑各个方面
```

- 过于笼统
- AI 无法判断候选人回答是否完整

### 3. JD 信息利用

AI 会在 `session.instructions` 中注入 JD 信息：

```python
instructions = f"""
...
岗位背景信息 (JD)：
岗位职责: {jd_data.get('responsibilities')}
任职要求: {jd_data.get('requirements')}
加分项: {jd_data.get('plus')}
...
"""
```

**作用**：
- AI 会结合 JD 评估候选人的匹配度
- 追问时会针对 JD 要求深入
- 最终评估会参考 JD 标准
