# 3.0 后端日志模组规范

## 1. 概述
为了方便排查实时面试过程中的连接问题、音频流问题以及 API 调用错误，系统引入了自动化的日志模组。该模组在每次后端程序启动时都会创建一个独立的日志文件。

## 2. 设计思路
- **独立性**：每次启动生成一个新的文件，避免单个日志文件过大，且方便按启动时间定位问题。
- **双重输出**：日志同时输出到控制台（Console）和文件（File）。
- **结构化**：包含时间戳、模块名、日志级别和具体消息。

## 3. 存储规范
- **目录**：`backend/logs/`
- **命名规则**：`server_YYYYMMDD_HHMMSS.log`
  - 例如：`server_20260310_103000.log` 表示 2026年3月10日 10:30:00 启动的会话。

## 4. 使用方法

### 4.1 引入日志
在 Python 模块中引入全局 logger 实例：
```python
from ..utils.logger import logger
```

### 4.2 记录日志
根据事件严重程度选择级别：
- `logger.info("...")`：常规流程（如连接成功、保存文件）。
- `logger.warning("...")`：异常但不影响流程（如无效 Token 尝试连接）。
- `logger.error("...")`：错误（如 API 调用失败、WebSocket 断开）。

## 5. 关键记录点
目前系统已在以下位置集成日志：
- **系统启动**：`main.py` 记录服务启动。
- **WebSocket 连接**：记录客户端连接、OpenAI Realtime 连接状态。
- **面试过程**：记录每题音频保存路径、AI 响应完成状态。
- **错误捕获**：记录中转过程中的所有异常。
