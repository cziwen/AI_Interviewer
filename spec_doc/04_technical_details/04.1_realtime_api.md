# Realtime 三段式 API 集成

## 当前实现（与代码一致）

系统不再把回复权交给上游端到端模型，而是：

- 上游仅负责 ASR
- 后端负责决策与文本生成
- 后端负责 TTS 合成与音频分片下发

对应实现入口：`backend/app/services/realtime/session_runner.py`

## ASR 接入模式

### 1) `sauc_v3`（推荐）

- URL：`wss://openspeech.bytedance.com/api/v3/sauc/bigmodel`
- 头：`X-Api-App-Key` / `X-Api-Access-Key` / `X-Api-Resource-Id` / `X-Api-Connect-Id`
- 协议：二进制帧（full request + audio request）

### 2) `openspeech_v2`

- URL：`wss://openspeech.bytedance.com/api/v2/asr`
- 头：`Authorization: Bearer;{token}`
- 协议：二进制帧

### 3) `gateway_json`

- URL：`wss://ai-gateway.vei.volces.com/v1/realtime?...`
- 协议：JSON 事件

## 后端回合执行流程

1. `_finalize_candidate_segment()` 等待转写
2. `_decide_next_turn()` 产出 `TurnPlan`
3. `_send_response_create_with_turn()`
4. `ResponseGenerator.generate()` 产生文本
5. `TtsSynthesizer.synthesize()` 产生 PCM16
6. `AudioChunker.iter_chunks()` 下发 `response.audio.delta`
7. 本地发送 `response.done` 并完成 turn

## 事件适配

- JSON 上游模式：`AsrAdapter.normalize_event`
- 二进制上游模式：`OpenSpeechAsrProtocol.parse_server_message`

统一内部事件：
- `speech_started`
- `speech_stopped`
- `transcript_completed`
- `input_committed`
- `error`

## 使用统计

- 文本 token 计入 `ARK_LLM_MODEL`
- TTS 音频时长计入 `ARK_TTS_MODEL`
- 候选人输入音频时长计入 `ARK_STT_MODEL`
