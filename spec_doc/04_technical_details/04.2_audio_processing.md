# 音频处理技术

## 📝 概述

本系统使用 PCM16 格式进行音频传输，采样率固定为 24kHz（由 ASR 上游语音网关 规定）。前端使用 Web Audio API 进行采集和播放，后端进行格式转换和存储。

## 🎛️ 音频格式规范

### PCM16 规格

| 参数 | 值 | 说明 |
|-----|---|------|
| **格式** | PCM16 (Linear PCM) | 16-bit 线性脉冲编码调制 |
| **采样率** | 24000 Hz | 固定值，由 ASR 上游语音网关 要求 |
| **声道数** | 1 (Mono) | 单声道 |
| **位深度** | 16 bit | 每个样本 2 字节 |
| **字节序** | Little Endian | 小端序 |
| **数据率** | 48000 bytes/s | 24000 × 2 |

### 数据表示

```
样本值范围：-32768 ~ 32767 (int16)
归一化范围：-1.0 ~ 1.0 (float32)

转换公式：
  float32 = int16 / 32768.0
  int16 = clamp(float32 * 32768, -32768, 32767)
```

## 🎤 前端音频采集

### 采集链路

```
麦克风 (MediaStream)
  ↓
AudioContext (24kHz)
  ↓
MediaStreamSource
  ↓
ScriptProcessorNode (2048 samples buffer)
  ↓ onaudioprocess 回调
Float32Array (归一化 -1.0~1.0)
  ↓
PCM16 转换 (int16 -32768~32767)
  ↓
Base64 编码
  ↓
WebSocket 发送
```

**为什么需要连接到 destination？**

部分浏览器（尤其 Safari）中，`ScriptProcessorNode` 必须连接到音频图的输出端（`destination`）才能保证 `onaudioprocess` 回调持续触发。使用 `gain=0` 的 Gain 节点可以在不产生声音的情况下保持连接。

#### 3. 音频处理回调

**代码位置**：[frontend/src/pages/Interview.tsx:232-269](../../frontend/src/pages/Interview.tsx#L232)

```typescript
processor.onaudioprocess = (e) => {
  if (ws.readyState === WebSocket.OPEN && audioContextRef.current) {
    const inputData = e.inputBuffer.getChannelData(0); // Float32Array

    // 1. 计算 RMS 音量（可视化用）
    let sum = 0;
    for (let i = 0; i < inputData.length; i++) {
      sum += inputData[i] * inputData[i];
    }
    const rms = Math.sqrt(sum / inputData.length);
    setVolume(rms);

    // 2. 半双工门控检查
    const now = audioContextRef.current.currentTime;
    const isAgentSpeaking = now < (nextStartTimeRef.current + 0.2);

    if (isAgentSpeaking) {
      return; // AI 正在说话，跳过发送
    }

    // 3. Float32 → PCM16 转换
    const pcm16 = floatTo16BitPCM(inputData);

    // 4. Base64 编码并发送
    const base64Audio = arrayBufferToBase64(pcm16);
    ws.send(JSON.stringify({ type: 'audio', audio: base64Audio }));
  }
};
```

#### 4. Float32 → PCM16 转换

**代码位置**：[frontend/src/pages/Interview.tsx:411-419](../../frontend/src/pages/Interview.tsx#L411)

```typescript
const floatTo16BitPCM = (input: Float32Array): ArrayBuffer => {
  const buffer = new ArrayBuffer(input.length * 2); // 每个样本 2 字节
  const view = new DataView(buffer);

  for (let i = 0; i < input.length; i++) {
    const s = Math.max(-1, Math.min(1, input[i])); // 限幅到 [-1, 1]
    const pcmValue = s < 0 ? s * 0x8000 : s * 0x7FFF; // int16 范围
    view.setInt16(i * 2, pcmValue, true); // true = Little Endian
  }

  return buffer;
};
```

#### 5. Base64 编码

**代码位置**：[frontend/src/pages/Interview.tsx:421-428](../../frontend/src/pages/Interview.tsx#L421)

```typescript
const arrayBufferToBase64 = (buffer: ArrayBuffer): string => {
  let binary = '';
  const bytes = new Uint8Array(buffer);
  for (let i = 0; i < bytes.byteLength; i++) {
    binary += String.fromCharCode(bytes[i]);
  }
  return btoa(binary); // Base64 编码
};
```

### 关键代码实现

#### 1. 音频队列管理

**代码位置**：[frontend/src/pages/Interview.tsx:430-494](../../frontend/src/pages/Interview.tsx#L430)

```typescript
const enqueueAudio = (base64Audio: string) => {
  if (!audioContextRef.current || !base64Audio) {
    console.warn('[TTS] Invalid audio or context');
    return;
  }

  // 1. Base64 → Binary
  let binary: string;
  try {
    binary = atob(base64Audio);
  } catch (e) {
    console.error('[TTS] Failed to decode base64:', e);
    return;
  }

  // 2. Binary → Uint8Array
  const bytes = new Uint8Array(binary.length);
  for (let i = 0; i < binary.length; i++) {
    bytes[i] = binary.charCodeAt(i);
  }

  // 3. Uint8Array → Int16Array (PCM16)
  if (bytes.byteLength % 2 !== 0) {
    console.warn('[TTS] Odd byte length:', bytes.byteLength);
  }
  const pcm16 = new Int16Array(bytes.buffer);

  // 4. Int16 → Float32 转换
  const float32 = new Float32Array(pcm16.length);
  for (let i = 0; i < pcm16.length; i++) {
    float32[i] = pcm16[i] / 32768.0;
  }

  console.log('[TTS] Decoded audio chunk: samples =', float32.length);

  // 5. 创建 AudioBuffer (24kHz)
  const audioBuffer = audioContextRef.current.createBuffer(1, float32.length, 24000);
  audioBuffer.getChannelData(0).set(float32);

  // 6. 创建 BufferSource 并调度播放
  const source = audioContextRef.current.createBufferSource();
  source.buffer = audioBuffer;
  source.connect(audioContextRef.current.destination);

  // 无缝衔接播放
  const startTime = Math.max(
    audioContextRef.current.currentTime,
    nextStartTimeRef.current
  );

  console.log('[TTS] Scheduling playback at', startTime, 'duration =', audioBuffer.duration);
  source.start(startTime);

  // 更新下次播放时间（用于半双工门控）
  nextStartTimeRef.current = startTime + audioBuffer.duration;
};
```

#### 2. 播放时间轴管理

```
时间轴示例（单位：秒）
─────────────────────────────────────────────────────>
0        1        2        3        4        5
│  Chunk1 (0.5s) │
│                │ Chunk2 (0.8s)  │
                               │ Chunk3 (0.6s) │
                                              │ ...

nextStartTimeRef: 0 → 0.5 → 1.3 → 1.9 → ...
```

**关键点**：
- 使用 `nextStartTimeRef` 追踪播放结束时间
- 新音频块总是从 `nextStartTimeRef` 开始播放
- 实现无缝衔接，避免间隙和重叠

## 🎧 后端音频处理

### PCM16 → WAV 转换

**代码位置**：[backend/app/api/realtime.py:23-33](../../backend/app/api/realtime.py#L23)

```python
import wave
import io

def pcm16_to_wav(pcm_data: bytes, sample_rate: int = 24000, channels: int = 1) -> bytes:
    """
    将原始 PCM16 数据封装为 WAV 格式。
    """
    with io.BytesIO() as wav_io:
        with wave.open(wav_io, 'wb') as wav_file:
            wav_file.setnchannels(channels)      # 单声道
            wav_file.setsampwidth(2)             # 16-bit = 2 bytes
            wav_file.setframerate(sample_rate)   # 24000 Hz
            wav_file.writeframes(pcm_data)
        return wav_io.getvalue()
```

### WAV 文件结构

```
WAV 文件 = RIFF Header + fmt Chunk + data Chunk

RIFF Header (12 bytes):
  0-3:   "RIFF"
  4-7:   文件大小 - 8
  8-11:  "WAVE"

fmt Chunk (24 bytes):
  12-15: "fmt "
  16-19: fmt chunk size (16)
  20-21: audio format (1 = PCM)
  22-23: channels (1)
  24-27: sample rate (24000)
  28-31: byte rate (48000)
  32-33: block align (2)
  34-35: bits per sample (16)

data Chunk (8 + data_size bytes):
  36-39: "data"
  40-43: data size
  44-...: 原始 PCM16 数据
```

## 📊 性能与资源消耗

### 数据量估算

**1 分钟音频的数据量**：

```
采样率：24000 Hz
位深度：16 bit = 2 bytes
时长：60 秒

原始 PCM16 大小：
  24000 × 2 × 60 = 2,880,000 bytes ≈ 2.75 MB

Base64 编码后：
  2.75 MB × 4/3 ≈ 3.67 MB

WAV 文件（含头部）：
  2.75 MB + 44 bytes ≈ 2.75 MB
```

**10 分钟面试估算**：

| 项目 | 大小 |
|-----|------|
| 候选人音频（10分钟） | ~27.5 MB (WAV) |
| AI 音频（5分钟估算） | ~13.8 MB (实时流) |
| WebSocket 传输总量 | ~60 MB (Base64) |

### 浏览器性能

- **内存占用**：AudioContext + Buffer ≈ 10-50 MB
- **CPU 占用**：`ScriptProcessorNode` ≈ 5-10%（单核）
- **网络带宽**：上传 ~400 KB/s，下载 ~200 KB/s

**优化建议**：
- 使用 `AudioWorklet` 替代 `ScriptProcessorNode`（现代浏览器）
- 定期释放已播放的 AudioBuffer

### 4. 常见问题

| 问题 | 原因 | 解决方案 |
|-----|------|---------|
| 听不到声音 | AudioContext suspended | 用户交互后调用 `audioContext.resume()` |
| 声音断续 | 缓冲区太小 | 增大 ScriptProcessorNode buffer size |
| 音量过小 | 麦克风增益低 | 检查系统设置或使用 GainNode |
| 音频失真 | 溢出截断 | 检查 floatTo16BitPCM 的 clamp 逻辑 |
| 采样率不匹配 | 使用了错误的采样率 | 确认所有地方都使用 24kHz |
