# 快速启动指南

## 🚀 5 分钟快速体验

### 前置条件
- Python 3.9+
- Node.js 16+
- OpenAI API Key

### 步骤 1：克隆项目
```bash
git clone <repository_url>
cd AI_Interviewer
```

### 步骤 2：后端设置
```bash
cd backend

# 创建虚拟环境（可选但推荐）
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
cat > .env << EOF
OPENAI_API_KEY=sk-your-api-key-here
DATABASE_URL=sqlite:///./ai_interview.db
ADMIN_USERNAME=admin
ADMIN_PASSWORD=admin123
EOF

# 启动后端
uvicorn app.main:app --reload
```

**提示**：服务启动后会显示 `Application startup complete`。

### 步骤 3：前端设置
```bash
# 新终端窗口
cd frontend

# 安装依赖
npm install

# 启动前端
npm run dev
```

### 步骤 4：访问应用
打开浏览器访问：http://localhost:5173

### 步骤 5：创建测试面试
1. 登录管理后台（默认账号：admin / admin123）
2. 点击"创建面试"
3. 填写候选人信息和岗位
4. 复制面试链接
5. 在新标签页打开链接，开始面试！

## 🔄 当前模式说明

**当前版本使用 OpenAI Realtime API 模式 (Voice Agent 增强版)**

**特点**：
- 🎯 **全自动对话**：基于 OpenAI Realtime VAD (语音活动检测)，候选人说完后 AI 会自动识别并即时回复，无需手动点击。
- 🎙️ **实时转写**：前端实时显示 AI 发言的文字转写，增强交互感知。
- ⚙️ **配置简单**：开箱即用，无需安装本地 STT/TTS 模型。
- 💸 **成本说明**：由 OpenAI 提供端到端的语音处理，产生 STT + TTS + LLM 费用。

**配置（必需）**：
```bash
# .env
OPENAI_API_KEY=sk-your-api-key-here
DATABASE_URL=sqlite:///./ai_interview.db
```

**可选环境变量（有默认值，可不配置）**：

- `ADMIN_USERNAME` / `ADMIN_PASSWORD`：后台默认管理员账号（代码默认 `admin` / `admin123`）
- `JWT_SECRET`：JWT 签名密钥（开发环境有默认值，生产环境务必自行修改）
- `UPLOAD_DIR`：面试录音等文件的保存目录（默认 `./app/static/uploads`，启动时会自动创建）
- `QUESTION_BANK_CSV_PATH` / `QUESTION_BANK_FIELD_MAP`：题库 CSV 路径及字段映射（仅在管理端题库功能中使用）

> 当前默认模式仅依赖 `backend/requirements.txt` 中已有的依赖即可运行，无需额外安装本地 STT/TTS 相关包。

## 💡 未来版本规划

后续版本将支持**本地 STT/TTS 模式**以降低成本：

- 使用 RealtimeSTT（Whisper）+ RealtimeTTS（Kokoro）等本地/自托管组件
- 成本有望降低 70%+
- 更高的可定制性

当前代码中已经包含部分本地语音相关实验模块：

- `backend/app/realtime/audio_module.py`：本地 TTS 引擎封装（Coqui、Kokoro、Orpheus）
- `backend/app/realtime/turndetect.py`：基于模型的说话人轮次检测

但这些模块**默认不会在启动流程中启用**，对应依赖也**未写入 `backend/requirements.txt`**。  
只要按照本快速启动文档配置，即会使用 OpenAI Realtime 的云端 STT/TTS，不会触发本地语音模块。

## ⚠️ 常见启动问题

### 问题 1：端口被占用
**症状**：`Address already in use`

**解决**：
```bash
# 查找占用进程
lsof -i :8000  # Linux/Mac
netstat -ano | findstr :8000  # Windows

# 杀死进程或更换端口
uvicorn app.main:app --reload --port 8001
```

### 问题 2：麦克风权限被拒绝
**症状**：浏览器无法访问麦克风

**解决**：
1. Chrome：设置 → 隐私和安全 → 网站设置 → 麦克风
2. 确保使用 HTTPS 或 localhost
3. 刷新页面重新授权

### 问题 3：OpenAI API 调用失败
**症状**：AI 无响应或报错

**解决**：
```bash
# 验证 API Key
curl https://api.openai.com/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"

# 检查余额
# 访问 https://platform.openai.com/usage

# 使用代理（可选）
export HTTPS_PROXY=http://127.0.0.1:7890
```

## 💡 性能优化建议

### 生产环境部署
```bash
# 使用 Gunicorn + Uvicorn Workers
gunicorn app.main:app \
  -w 4 \
  -k uvicorn.workers.UvicornWorker \
  --bind 0.0.0.0:8000

# 使用 Nginx 反向代理
# 配置 WebSocket 支持
proxy_http_version 1.1;
proxy_set_header Upgrade $http_upgrade;
proxy_set_header Connection "upgrade";
```

### 数据库迁移（可选）
```bash
# 切换到 PostgreSQL
DATABASE_URL=postgresql://user:pass@localhost/ai_interview
```

### 缓存优化
```bash
# Redis 缓存 LLM 响应（可选）
pip install redis
```
