# 5.0 岗位配置指南 (Job Profile Configuration)

本指南详细说明了如何为不同岗位配置 CSV 题库和 JSON 岗位描述 (JD)，以实现个性化的 AI 面试流程。

## 1. 岗位配置概述

每个岗位由两个核心文件组成：
1.  **CSV 题库文件**：定义面试中可能被问到的所有主问题及其参考方向。
2.  **JSON JD 文件**：定义岗位的背景信息（职责、要求）以及面试的控制参数（问题数、追问限额等）。

通过管理接口 `POST /api/job_profiles/` 上传这两个文件，并指定一个唯一的 `position_key`（如 `senior_backend_engineer`）。

---

## 2. CSV 题库格式规范

CSV 文件必须包含两列，且表头（Header）必须为 `question` 和 `reference`。

| 列名 | 说明 | 是否必填 |
| :--- | :--- | :--- |
| `question` | 面试官将要提问的主问题文本。 | 是 |
| `reference` | **答案的指导方向或参考要点**。AI 面试官会根据此内容判断候选人回答是否完整，并决定是否进行追问。 | 否 |

### 示例文件 (`questions.csv`)：
```csv
question,reference
"请介绍一下你在分布式系统设计中的经验。","重点关注：CAP定理的权衡、服务拆分原则、数据一致性方案（如两阶段提交或最终一致性）。"
"如何处理高并发场景下的数据库写压力？","参考方向：数据库分库分表、引入消息队列削峰填谷、使用缓存预热、读写分离架构。"
"谈谈你对微服务架构中服务治理的理解。","开放题，关注：服务注册发现、熔断限流、链路追踪、配置中心。"
```

---

## 3. JSON JD 格式规范

JSON 文件用于定义岗位背景和面试行为参数。

### 核心字段说明：

| 字段名 | 类型 | 说明 | 默认值 |
| :--- | :--- | :--- | :--- |
| `main_question_count` | Integer | 本场面试从题库中**随机抽取**的主问题数量。 | 3 |
| `followup_limit_per_question` | Integer | 每个主问题之后允许的最大追问次数。 | 1 |
| `expected_duration_minutes` | Integer | 告知 AI 面试官整场面试的预期时长（分钟）。系统会据此动态控制面试节奏和超时收尾行为。 | 10 |
| `responsibilities` | String | 岗位职责描述，帮助 AI 理解业务背景。 | - |
| `requirements` | String | 任职要求描述，帮助 AI 判断技术深度。 | - |
| `plus` | String | 加分项描述。 | - |

### 示例文件 (`jd.json`)：
```json
{
  "main_question_count": 3,
  "followup_limit_per_question": 1,
  "expected_duration_minutes": 15,
  "responsibilities": "负责高并发后端系统的架构设计与核心代码编写，参与技术选型和团队技术指导。",
  "requirements": "5年以上 Java/Go 开发经验，精通分布式架构，熟悉 Kubernetes 和云原生技术。",
  "plus": "有开源项目贡献经验或大规模系统迁移经验者优先。"
}
```

---

## 4. 配置生效流程

1.  **上传配置**：调用 `POST /api/job_profiles/`，上传 `jd_file` 和 `question_csv`。
2.  **创建面试**：在创建面试请求 (`POST /api/interviews/create`) 中，传入对应的 `position_key`。
3.  **动态生成**：
    *   后端会根据 `main_question_count` 从 CSV 中**随机抽取**题目。
    *   AI 面试官的 `session.instructions` 会被自动注入 JD 中的背景和每道题的 `reference`。
4.  **面试执行**：AI 面试官将严格遵守 JD 中定义的提问数量和追问限额。系统会实时监控面试进度，若超时则自动触发自然收尾逻辑。若严重超时（超过 20% 或 5 分钟，取最大值），系统将强制结束面试。

---

## 5. 常见问题 (FAQ)

*   **Q: 如果 CSV 里的题目少于 `main_question_count` 怎么办？**
    *   A: 系统会自动取实际题目数量，不会报错。
*   **Q: `reference` 必须是标准答案吗？**
    *   A: 不必。它更多是给 AI 的“小抄”，告诉它应该引导候选人往哪些方向回答。
*   **Q: 可以不上传文件直接创建面试吗？**
    *   A: 可以。如果不提供 `position_key`，系统将回退到原有的基于岗位名称自动生成题目的逻辑。
